# Phase 2-4: ブランド検索 — 設計メモ

**目的**: 「VASTDIGI って、聞いたことないけど大丈夫？」に、公開データだけで答える。
**原則**: AIの推測ではなく、**照会可能な一次情報**を並べる。断定はしない（ツール本体と同じ）。

## データソース調査結果（2026-07-10 実測）

| ソース | 認証 | 即使用 | 何がわかるか | 実測 |
|---|---|---|---|---|
| **Wikipedia API** | 不要 | ✅ | ブランドの社会的認知の有無 | VASTDIGI=0件／キオクシア=168件。**判別力あり** |
| **公式サイトの実在** | 不要 | ✅ | ブランド名でのWeb検索上位に公式ドメインがあるか | 実装可（`web_search` 相当 or クライアント側リンク生成） |
| **J-PlatPat（商標）** | 不要 | △ | 商標登録の有無 | サイトは200。**スクレイピングは規約上避ける** → 検索画面への直リンク生成に留める |
| **国税庁 法人番号API** | **要アプリケーションID** | ❌ | 日本法人としての実在・所在地 | **発行に1〜1.5か月**。無料・有効期限なし。**今すぐ申請すべき** |

## 設計判断

### 1. 法人番号APIは「今日申請、来月実装」
発行リードタイムがあるため、**Phase 2-4 の実装を待たずに今すぐ申請する**（申請自体はコスト0）。
届いたら「日本国内の販売元」に対して登記の実在確認を追加できる。

- 申請先: 適格請求書発行事業者公表サイトのアプリケーションID発行フォーム
- その後、国税庁 軽減税率・インボイス制度対応室へ「法人番号システムWeb-APIのみ利用希望」とメール
- 参考: https://www.houjin-bangou.nta.go.jp/webapi/index.html

### 2. 先行してリリースできるのは「Wikipedia + 公式サイト + 商標リンク」
法人番号を待たずに価値が出る。特にWikipediaヒット数は、無名ブランドの検出に実測で効いた。

### 3. スコアに直結させず、「調べる材料」として並べる
ブランドの知名度は品質と無関係（優れた無名ブランドはある）。
→ **確認レベルの加点・減点には使わない**。「ブランドを調べる」という独立セクションとして結果カードの下に置く。
これは消費者教育というプロダクトの筋にも合う（点数を出すのではなく、調べ方を教える）。

## UI 案

結果カードの下に、折りたたみで:

```
▸ ブランド「VASTDIGI」を調べる
  ・Wikipedia: 該当する項目は見つかりませんでした
  ・公式サイトらしきページ: 見つかりませんでした
  ・商標を検索する（J-PlatPat）→ [リンク]
  ・法人番号で調べる（国税庁）→ [リンク]        ※API取得後は実際の照合結果に置き換え
```

ブランド名は、貼り付けテキストの「ブランド表記: ○○のストアを表示」から抽出できる（ブックマークレット・拡張は既に収集済み）。

## 実装メモ

- Wikipedia API は CORS 許可済み（`origin=*`）→ **クライアント側から直接叩ける**。
  サーバーにブランド名を送らずに済むので、プライバシー設計と整合する。
  ```
  https://ja.wikipedia.org/w/api.php?action=query&list=search&srsearch=<brand>&format=json&origin=*
  ```
- J-PlatPat は検索URLを組み立ててリンクするのみ（スクレイピングしない）
- 公式サイト実在確認は、サーバー側で `web_search` を叩くとブランド名がログに乗る懸念 →
  当面は「Google検索へのリンク生成」に留め、実在判定はユーザーの目に委ねる（教育的にも正しい）
